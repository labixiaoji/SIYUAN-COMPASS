# 修改方案 v2

本文档基于第一轮测试反馈整理，目标是把修改拆成可执行、可测试、风险可控的阶段。

建议不要一次性全部修改。优先先解决用户完成率和报告生成体验，再逐步扩展问卷字段和报告结构。

---

## 阶段一：提交体验与填写反馈

### 目标

解决以下问题：

- 用户填完后不知道是否提交成功；
- 报告生成中缺少明确反馈；
- 生成失败后不知道怎么处理；
- 漏填题目没有明确提醒；
- 用户重复填写、重复提交。

### 修改内容

#### 1. 提交后明确状态

提交后展示清晰状态：

```text
问卷已提交
正在生成画像
正在生成报告
生成成功
生成失败
```

前端已有生成任务接口和进度轮询，建议进一步强化文案和视觉提示。

#### 2. 防重复提交

用户点击提交后：

- 禁用提交按钮；
- 显示“生成中”；
- 禁止再次点击提交；
- 如果生成失败，再恢复可操作状态。

#### 3. 生成失败可重试

失败时展示：

- 失败阶段；
- 失败原因；
- 重新生成按钮；
- 返回修改问卷按钮。

#### 4. 漏填提醒

提交前检查必填项：

- 必填题显示 `*`；
- 漏填题红框或高亮；
- 自动滚动到第一道漏填题；
- 顶部显示“还有 N 项未完成”；
- 后端返回的校验错误要映射到具体字段。

#### 5. 草稿自动保存

问卷内容保存到浏览器 `localStorage`。

刷新或重新进入问卷页时提示：

```text
检测到未提交草稿，是否恢复？
```

提交成功后清除草稿。

### 涉及文件

```text
frontend/src/pages/AssessmentPage.tsx
frontend/src/styles/global.css
backend/app/services/assessment_validator.py
backend/app/api/assessments.py
```

### 验收标准

- 漏填时能看到具体漏填题；
- 提交后不会重复提交；
- 生成过程中能看到阶段变化；
- 生成失败后有明确重试入口；
- 刷新页面后能恢复草稿；
- 成功生成报告后草稿被清除。

## 阶段二：基础信息与动态问卷逻辑

### 目标

让问卷适配不同学历阶段，避免本科生、硕士生、博士生看到不相关问题。

### 修改内容

#### 1. 增加基础信息

新增字段：

```text
姓名
学校
学号或用户编号
联系方式
```

同时增加隐私说明：

```text
以上信息仅用于报告归属、测试回访和系统优化，不会用于公开展示。
```

#### 2. 增加学历阶段

先选择：

```text
本科
硕士
博士
```

#### 3. 年级动态展示

根据学历阶段展示年级。

本科：

```text
大一
大二
大三
大四
```

硕士：

```text
研一
研二
研三及以上
```

博士：

```text
博一
博二
博三
博四及以上
```

#### 4. 升学问题动态展示

本科生显示：

```text
已保研
正在准备考研
准备出国读研
直接就业
考公或考编
尚未确定
```

硕士生显示：

```text
是否考虑读博
是否准备就业
是否考虑考公考编
是否考虑出国
尚未确定
```

博士生不再展示读硕、读博问题，改为询问：

```text
高校或科研院所
企业研发
博士后
公务员或事业单位
创业或自由职业
其他发展方向
```

### 涉及文件

```text
backend/app/schemas/assessment.py
frontend/src/types/assessment.ts
frontend/src/pages/AssessmentPage.tsx
frontend/src/features/assessment/options.ts
backend/app/services/assessment_validator.py
backend/app/services/question_rules.py
backend/app/services/profile_prompt.py
```

### 验收标准

- 本科、硕士、博士看到不同年级选项；
- 硕士生不再被问“是否读硕士”；
- 博士生不再被问“是否读硕士/读博士”；
- 大四本科生读硕问题中包含“已保研”；
- 基础信息能保存到数据库；
- 报告和管理员后台能区分学生身份。

## 阶段三：补充画像判断题

### 目标

提升用户画像准确度，减少 AI 只顺着用户愿景写的情况，让系统能根据证据给出判断。

### 修改内容

#### 1. 学业竞争力

新增字段：

```text
当前 GPA
GPA 满分
专业排名
专业总人数
英语成绩或证书
科研、竞赛、论文经历
是否挂科或重修
```

用途：

- 判断保研可能性；
- 判断考研、出国、读博可行性；
- 判断学术路径和职业路径的匹配度。

#### 2. 第二专业

新增字段：

```text
是否修读第二专业
第二专业名称
修读程度
是否考虑从事第二专业相关方向
```

用途：

- 识别复合型发展路径；
- 为 Plan B 或 Plan C 提供依据。

#### 3. 转专业经历

新增字段：

```text
是否转过专业
原专业名称
转专业原因
原专业相关知识和能力是否仍然保留
```

用途：

- 判断跨专业能力；
- 判断旧专业是否可以成为备选路径。

#### 4. 优势特质与成果证据

新增问题：

```text
从小到大，你最常被他人称赞的特质是什么？
```

提示示例：

```text
记忆力强
动手能力强
表达能力强
逻辑清晰
反应迅速
组织协调能力强
有领导力
审美能力强
擅长某项才艺或技能
```

继续追问：

```text
你是否凭借这些特质取得过相关成果？
```

例如：

```text
获奖
证书
项目成果
作品
组织活动经历
```

#### 5. 兴趣探索

新增问题：

```text
哪些事情能让你长时间沉浸其中？
你喜欢学习哪方面的知识？
哪些事情即使没有外部奖励，你也愿意主动完成？
你更喜欢分析问题、与人沟通、动手实践还是组织管理？
```

#### 6. 坚持力与执行力

长期坚持能力：

```text
为实现目标，在没有监督、且一年内看不到成果的情况下，你能否坚持？
```

建议使用五级量表。

执行能力：

```text
A. 执行力较强，确定要做的事情通常能够完成
B. 执行力一般，重要事情能够推进，但容易受状态影响
C. 执行力较弱，即使知道事情重要，也经常难以启动
```

建议增加实际案例说明，避免完全依赖主观评价。

#### 7. 抗压能力

新增问题：

```text
遇到失败后通常多久恢复？
面对他人负面评价时，你的典型反应是什么？
是否容易产生自我怀疑？
是否能够主动寻找解决办法？
是否需要他人安慰才能恢复？
```

可选项示例：

```text
需要一周以上恢复
需要几天恢复
一天左右可以恢复
能迅速调整并寻找解决办法
基本不受影响
```

#### 8. 高强度工作承受能力

新增问题：

```text
过去是否有过连续一个月以上高强度投入的经历？
```

选项：

```text
有过，并且能够较好承受
有过，但只能短期承受
有过，对身体或情绪影响较大
没有类似经历
不确定自己是否能够承受
```

#### 9. 事务性工作接受程度

新增问题：

```text
你能否接受纯事务性、琐碎、繁杂的工作？
```

选项：

```text
可以接受，并且擅长处理细节
短期可以接受
只接受部分事务性工作
不喜欢重复、琐碎的工作
完全无法接受
```

#### 10. 职业风险偏好

让用户在三类职业环境中选择或排序：

```text
A. 铁饭碗，收入和职业发展按部就班，规章制度严格
B. 较为宽松，发展更多取决于个人能力与行业环境，但有失业风险
C. 完全自主，但收入波动巨大，例如创业或自由职业
```

#### 11. 准备事项详细说明

在“为了未来目标，你已经做过哪些准备？”后增加填空题：

```text
请具体展开说说：
修读过哪些课程？
参加过哪些比赛？
做过哪些科研项目？
取得过哪些荣誉或技能证书？
有哪些个人作品？
有哪些相关实习？
参加过哪些学生社团或学生工作？
```

### 涉及文件

```text

frontend/src/types/assessment.ts
frontend/src/pages/AssessmentPage.tsx
frontend/src/features/assessment/options.ts
backend/app/services/assessment_validator.py
backend/app/services/question_rules.py
backend/app/services/profile_prompt.py
backend/app/services/report_prompt.py
```

### 验收标准

- 新增字段能正常填写、提交、保存；
- 必填和非必填边界清晰；
- AI 画像能引用新增证据；
- 报告中能体现特质、成果、执行力、抗压、风险偏好；
- 不因新增字段导致报告明显变长或失控。

## 阶段四：报告结构升级

### 目标

让报告更直接回应学生困惑，并提供不止两条路径。

### 修改内容

#### 1. 增加 Plan C

报告路径改为：

```text
Plan A：主攻路径
Plan B：备选路径
Plan C：系统建议路径
```

Plan C 的定位：

- 跳出学生原有设定；
- 基于系统识别的优势、兴趣、限制和风险给出第三条值得探索的路径；
- 尤其适合“自己也不知道想干什么”的学生。

#### 2. 固定回应最大困惑

报告中增加固定内容：

```text
你现在最大的困惑是什么？
这个困惑背后的真正问题是什么？
接下来可以如何验证？
```

要求：

- 必须引用用户选择的 `careerConfusions`；
- 必须回应 `mainConfusionText`；
- 不允许只泛泛鼓励。

#### 3. 报告字数调整

加入 Plan C 后，报告建议控制在：

```text
5000—6000字
```

同时每个模块继续限制篇幅，避免内容过长。

#### 4. 强化安全提醒

继续保留安全提醒：

```text
本报告不是医学诊断、心理诊断或人生终局结论。
所有分析和建议仅供规划时参考，不构成唯一决定。
```

#### 5. 强化评分入口

报告页底部明显展示：

```text
请为本报告评分
```

管理员后台继续展示：

- 平均理解度；
- 平均启发度；
- 平均行动性；
- 平均推荐度；
- 低分报告列表。

### 涉及文件

```text
backend/app/schemas/profile.py
backend/app/services/profile_prompt.py
backend/app/services/report_prompt.py
backend/app/services/report_quality_check.py
frontend/src/components/ReportRenderer.tsx
frontend/src/pages/ReportPage.tsx
frontend/src/pages/FeedbackPage.tsx
frontend/src/pages/AdminPage.tsx
```

### 验收标准

- 报告包含 Plan A、Plan B、Plan C；
- Plan C 不是 Plan A/B 的重复；
- 报告明确回应最大困惑；
- 报告字数稳定在 5000—6000 字；
- 报告底部评分入口明显；
- 管理员后台能看到反馈评分。

## 特别注意：薪资与住房问题

之前已经决定：画像和报告不分析薪资。

因此，关于住房和居住状态，建议只用于理解：

```text
城市偏好
通勤偏好
租房/自有住房偏好
长期定居意愿
家庭生活安排
```

不建议让 AI 判断：

```text
收入是否现实
购房能力
薪资区间
住房购买力
```

否则会回到第一轮中“AI 对薪资判断不准确”的问题。

## 推荐执行顺序

```text
第 1 步：阶段一
修提交状态、漏填提醒、草稿保存。

第 2 步：阶段二
加基础信息、学历阶段、动态跳题。

第 3 步：阶段三
加 GPA、二专、转专业、特质、兴趣、执行力、抗压、风险偏好。

第 4 步：阶段四
升级画像和报告结构，加入 Plan C，强化最大困惑回应。
```

---

## 版本验收建议

每完成一个阶段，都至少做以下测试：

```text
1. 新用户注册登录
2. 填写完整问卷
3. 漏填提交测试
4. 刷新恢复草稿测试
5. 生成画像
6. 生成报告
7. 查看报告
8. 提交评分
9. 管理员后台查看记录
10. 管理员编辑报告
```

每一阶段单独测试通过后，再进入下一阶段。
